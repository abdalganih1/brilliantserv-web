## 📊 نظام البيانات (Hybrid JSON Loading)

### كيف يعمل؟

الموقع يستخدم **نظام تحميل هجين** للبيانات:

1. **مع سيرفر (استضافة)**: يحمّل البيانات من ملفات JSON المنفصلة
2. **محلياً (file:///)**: يستخدم البيانات المضمنة في HTML كـ fallback

```javascript
// Hybrid: Try fetch, fallback to embedded
try {
    const response = await fetch('media/projects.json');
    data = await response.json();
    console.log('✓ Loaded from JSON');
} catch (e) {
    data = embeddedData;
    console.log('✓ Using embedded data (file:// mode)');
}
```

### 3. استكشاف الأخطاء وإصلاحها (Troubleshooting Log)

#### أ) مشكلة Git Bash و Deploy.bat
*   **المشكلة**: عند تشغيل `ssh_connection/deploy.bat` من طرفية Git Bash، ظهر خطأ `command not found` أو `path not found`.
*   **السبب**: ملفات `.bat` مخصصة لبيئة Windows CMD ولا تفهمها Git Bash بشكل مباشر بنفس مسارات Windows (Backslashes `\`).
*   **الحل**:
    *   **للمرات القادمة**: شغل الملف من `cmd` أو `PowerShell` مباشرة: `ssh_connection\deploy.bat`.
    *   **الحل الآني**: قمنا بتنفيذ أوامر `git push` يدوياً ثم أمر `ssh` للاتصال بالسيرفر.

#### ب) تكرار المحتوى أثناء النشر
*   **المشكلة**: وجود ملفات `.html` قديمة (مثل `blog/leak-detection-jeddah.html`) بجانب النسخ الجديدة `.php`.
*   **الحل الجذري**: تحديث `templates/template-3/deploy.sh` (على السيرفر) ليقوم بمسح ملفات `.html` القديمة تلقائياً من مجلد `blog/` قبل نسخ الملفات الجديدة، لضمان نظافة السيرفر.
